# 阶段 1.1：Program Counter（PC）

PC（程序计数器）是处理器/GPU 中最基础的状态之一。它保存“下一步要执行的位置”。当前项目尚未取指，因此 PC 暂时只是一个每拍加一的 32 位计数器。

## 1. PC 模块

`src/pc.sv` 定义独立模块：

```text
        ┌───────────┐
clk ───►│           │
rst ───►│    pc     │───► pc_value[31:0]
        └───────────┘
```

端口：

| 端口 | 含义 |
| --- | --- |
| `clk` | 时钟；PC 只在上升沿改变 |
| `rst` | 同步复位；为 1 时将 PC 清零 |
| `pc_value[31:0]` | 32 位当前 PC 值 |

核心规则：

```systemverilog
if (rst) begin
    pc_value <= 32'd0;
end else begin
    pc_value <= pc_value + 32'd1;
end
```

## 2. 为什么是 32 位

`logic [31:0] pc` 表示一个 32 位无符号数：

```text
最小值：0
最大值：2³² - 1
```

现在它是计数器；以后它会表示指令地址。当前每个非复位时钟上升沿加 `1`，便于学习。真实处理器的 PC 增量会取决于指令宽度，分支或跳转还会改写 PC。

## 3. 时序行为

PC 是同步复位寄存器，因此只在时钟上升沿变化：

| 上升沿时的 `rst` | 上升沿后 PC |
| ---: | --- |
| `1` | `0` |
| `1` | `0` |
| `0` | 前一值 `+ 1` |
| `0` | 前一值 `+ 1` |

例如，下面的每一列竖线都是同一个时刻；PC 的新值在对应上升沿后的稳定阶段显示：

```text
时间 →              t0          t1          t2          t3          t4          t5

clk = 1          ┌────┐     ┌────┐     ┌────┐     ┌────┐     ┌────┐     ┌────┐
clk = 0  ────────┘    └─────┘    └─────┘    └─────┘    └─────┘    └─────┘    └──
                    ↑          ↑          ↑          ↑          ↑          ↑

rst = 1  ────────────────────┐                                      ┌──────────
rst = 0                      └──────────────────────────────────────┘

pc[31:0]          0          0          1          2          3          0
```

在 `t0`、`t1` 时，`rst=1`，所以 PC 保持 `0`。在 `t2` 之前复位已释放，因此 `t2` 的上升沿后 PC 从 `0` 变为 `1`，随后依次变为 `2`、`3`。在 `t5` 前重新拉高复位，`t5` 的上升沿后 PC 回到 `0`。

## 4. 顶层集成

`src/gpu_top.sv` 实例化 PC：

```systemverilog
pc program_counter (
    .clk(clk),
    .rst(rst),
    .pc_value(pc)
);
```

- 第一个 `pc`：模块类型，来自 `module pc`。
- `program_counter`：这个模块实例的名字。
- `.clk(clk)`：把顶层 `clk` 连接到实例的 `clk` 端口。
- `.pc_value(pc)`：把实例的 `pc_value` 输出连接到顶层公开的 `pc` 输出。

PC 的状态只由 `src/pc.sv` 保存和更新；`gpu_top` 只负责连接模块。

## 5. 两层测试

项目使用两类测试：

| 测试 | 顶层模块 | 验证内容 |
| --- | --- | --- |
| `test/test_pc.py` | `pc` | PC 模块本身的复位、保持与连续递增 |
| `test/test_reset.py` | `gpu_top` | 顶层的 `initialized` 与 PC 接线是否正确 |

两层测试的好处是：若 PC 计数错误，可先看独立模块；若独立模块正确、顶层测试失败，则问题通常在实例化或端口连接。

测试仍遵循阶段 0 的规则：在下降沿改变 `rst`，在上升沿后的 `ReadOnly()` 阶段读取 PC，确保读取的是非阻塞赋值完成后的稳定值。

## 6. 运行

```bash
source .venv/bin/activate
make test
```

该命令会分别仿真 `gpu_top` 与 `pc`，避免不同硬件顶层共用构建产物。
